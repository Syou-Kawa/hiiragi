# 画面仕様書(使い方ガイド)

ポートフォリオサイト(Next.js)の各画面の仕様と使い方をまとめたもの。
画面の実体は `hiiragi-portfolio-site/frontend/app/` 配下にある。起動方法は [local-setup.md](local-setup.md) を参照。

| URL | 画面 | 種別 |
|---|---|---|
| `/` | トップ | 公開(Server Component・API から作品取得) |
| `/about` | 自己紹介 | 公開(静的) |
| `/works` | 制作実績 | 公開(Server Component・API から作品取得) |
| `/service` | サービス内容・料金 | 公開(静的) |
| `/contact` | お問い合わせ・ご依頼 | 公開(Client Component・フォーム送信あり) |
| `/admin` | 作品管理 | 管理者用(パスワード認証) |

---

## トップ(`/`)

サイトの入り口。上から順に以下のセクションが並ぶ。

1. **ヒーロー** … アイコン・名前・キャッチコピー。ボタン3つ
   - 「お問い合わせ」→ `/contact`
   - 「制作のご依頼」→ `/contact?kind=request`(ご相談内容が「Webサイト制作のご依頼」で初期選択された状態で開く)
   - 「作品を見る」→ `/works`
2. **About** … 自己紹介の要約と `/about` への導線
3. **Works** … 最新の作品を **3件だけ** 表示(管理画面の並び順の上から3件)。「すべての作品を見る」で `/works` へ
4. **Service** … サービスの概要と `/service` への導線
5. **CTAバンド** … お問い合わせへの誘導
6. **Follow Me** … SNS リンクアイコン

### 使い方・運用メモ

- 作品は毎回 API から取得(`cache: "no-store"`)するため、管理画面で保存すると **リロードだけで即反映** される
- API(Laravel)が停止していても画面自体は表示される(Works セクションが空になるだけ)
- ヒーローのアイコン画像は `api/public/images/icon.svg` を差し替えると変わる

## 自己紹介(`/about`)

プロフィール・経歴・実績・スキルを載せた静的ページ。

- 内容(経歴のタイムライン、実績件数、スキルタグなど)は DB ではなく **`app/about/page.tsx` に直接書かれている**。変更したいときはこのファイルを編集する
- プロフィール画像はトップと同じ `api/public/images/icon.svg` を参照している。差し替えはファイル置き換えで行う
- ページ末尾に `/contact` への CTA バンドあり

## 制作実績(`/works`)

登録済みの作品を全件表示するギャラリーページ。

### 使い方(閲覧者)

- **カテゴリ絞り込み**: 上部のボタン(「すべて」+登録済みカテゴリ)をクリックすると表示が切り替わる
- **詳細表示**: カードをクリックするとモーダルが開き、画像・カテゴリ・タイトル・説明文(全文・改行あり)が見られる。「✕」またはモーダル外クリックで閉じる
- カード上では説明文は **1行目だけ** 表示される(全文はモーダルで)

### 運用メモ

- 作品の追加・編集・並び替えはすべて `/admin` で行う。表示順は管理画面の並び順そのまま
- カテゴリボタンは登録作品のカテゴリから自動生成される(カテゴリのマスタ管理は不要)
- API が停止していると「作品を読み込めませんでした。バックエンド(Laravel)が起動しているかご確認ください。」と表示される

## サービス内容・料金(`/service`)

料金プランと依頼の流れを載せた静的ページ。

- **料金プラン3種**(Webサイト制作 ¥150,000〜 / LP制作 ¥80,000〜 / バナー制作 ¥5,000〜)。各カードの「相談してみる」は `/contact?kind=request` へ遷移
- **ご依頼の流れ5ステップ**(お問い合わせ → ヒアリング・お見積もり → ご契約・制作開始 → ご確認・修正 → 納品・お支払い)
- プラン内容・価格・流れの文言は `app/service/page.tsx` 冒頭の `PLANS` / `FLOW` 定数を編集して変更する(DB 管理ではない)

## お問い合わせ(`/contact`)

閲覧者が依頼・相談を送るフォーム。

### 使い方(閲覧者)

1. お名前・メールアドレス・ご相談内容(5種から選択)・メッセージを入力(すべて必須)
2. 「送信する」をクリック → `POST /api/contact` に送信
3. 成功すると「✿ 送信しました」の完了画面に切り替わる。失敗時はエラーメッセージが出るので再送するか SNS の DM から連絡

- `?kind=request` 付きで開くと、ご相談内容が「Webサイト制作のご依頼」で初期選択される(トップやサービスページの「制作のご依頼」ボタン経由)

### 運用メモ

- **ローカル環境ではメールは実際に送信されない**(`MAIL_MAILER=log`)。送信内容は `api/storage/logs/laravel.log` に書き出されるので、動作確認はログを見る
- 本当にメールを届かせる SMTP 設定は [hiiragi-portfolio-site/SETUP.md](../hiiragi-portfolio-site/SETUP.md) を参照

## 作品管理(`/admin`)

作品の登録・編集・削除・並び替えを行う管理者専用画面。1ページで「ログイン → 作品管理」が完結する。

### ログイン

1. `http://localhost:3000/admin` を開く
2. パスワード(`api/.env` の `ADMIN_PASSWORD` の値)を入力して「ログイン」
3. 成功するとトークンが `localStorage`(key: `admin_token`)に保存され、作品管理画面に切り替わる。次回以降はログイン不要(ログアウトするか、トークンが無効になるまで)

- パスワードを間違えると「パスワードが違います」と表示される
- 右上の「ログアウト」でトークンを破棄してログイン画面に戻る
- 管理 API がトークン無効(401)を返した場合も自動でログイン画面に戻る

### 作品の登録

1. 上部のドロップゾーンに画像を **ドラッグ&ドロップ**(またはクリックしてファイル選択)。JPG / PNG / GIF / WebP、4MB まで。プレビューが表示される
2. 作品タイトル(必須)・カテゴリ(必須)・説明文(任意・改行OK)を入力
   - カテゴリは自由入力だが、既存カテゴリが入力候補(datalist)に出るので表記ゆれを防げる
   - 説明文の1行目が一覧カードに表示され、全文はモーダルに表示される
3. 「✿ 作品を追加」をクリック → 保存されると即座に公開側に反映される

- 新規登録では画像が必須(未選択だと「画像をドラッグ&ドロップしてください」と警告)

### 作品の編集

1. 下部の「登録済みの作品」一覧で対象の「編集」をクリック → フォームに内容が読み込まれ、ページ上部へスクロールする
2. 内容を修正して「✿ この内容で更新」をクリック
   - 画像は差し替えたいときだけ新しくドロップすればよい(**ドロップしなければ現在の画像を維持**)
3. やめる場合は「キャンセル」でフォームがクリアされる

### 削除・並び替え

- **削除**: 一覧の「削除」→ 確認ダイアログで OK。元に戻せないので注意
- **並び替え**: 一覧の「↑」「↓」で1つずつ移動。操作した瞬間に保存される(保存ボタンなし)。この並び順がそのまま公開側の表示順になり、トップページには **上から3件** が表示される

### 補足

- `/admin` の URL 自体は誰でも開けるが、パスワード+API トークンで保護されている。URL を隠したい場合は `app/admin` のフォルダ名を変えるだけでよい
- パスワードの変更は `api/.env` の `ADMIN_PASSWORD` を書き換えて API を再起動する
